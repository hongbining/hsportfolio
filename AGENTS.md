# Project Philosophy

This portfolio represents an experienced enterprise software engineer.

Everything should communicate reliability, engineering quality, and maintainability.

Never optimize for visual effects over clarity.

Think like a senior engineer working on a production SaaS product.

## Coding Rules

- TypeScript strict only
- Server Components by default
- Client Components only when interaction is required
- Reusable components
- Accessibility first
- Semantic HTML
- Mobile-first
- Tailwind CSS v4 only
- No inline styles
- No duplicated code
- No unnecessary dependencies
- Prefer composition over inheritance

## UI Rules

Inspired by: Vercel, Stripe, Linear, shadcn/ui.

Avoid: portfolio templates, heavy gradients, glassmorphism everywhere, excessive
animations, skill percentage bars, progress circles.

## Content Rules

Do not use generic portfolio sections. Always focus on: Business Problem,
Architecture, Technical Decisions, Trade-offs, Troubleshooting, Lessons Learned.

Projects should feel like engineering case studies.

## Target Audience

CTO, Engineering Manager, Tech Lead. The website should convince them that the
author can own a product from architecture to production.
